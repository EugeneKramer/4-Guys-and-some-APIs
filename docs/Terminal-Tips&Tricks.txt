                               \    GENERAL    /
                                ~~~~~~~~~~~~~~~

ls/dir ~ lists current directory(folder) contents

ls -la ~ lists detailed file info for currenct directory

pwd ~ current file path

cd name ~ changes to selected directory

cd .. ~ moves up one directory

cd - ~ go back to previous directory (*quickswitch)

cd /drive letter ~ changes the selected drive (ex. cd /c   cd /e)

find ~ list ALL files in the working dir. extremely powerful for searching. more info - https://danielmiessler.com/study/find/ 

find | grep "test string" ~ searches for files and folders with the string

grep -ir "text string" ~ search EVERYTHING in the  current dir and sub dirs for a string of text

cat "filename" ~ show the contents of a file

history ~ show all recently used commands

mkdir foldername ~ creates new directory in current directory

mkdir foldername name name ~ creates multiple directories at once

mkdir -p foldername/subfolder/subfolder/etc  ~ makes one folder with subfolders

touch filename.ext ~ creates a file with that extension (ex. index.html)

touch filename.ext name.ext name.ext ~ creates multiple files at once

mkdir name && touch thatdirname/file.ext ~ makes directory with file (ex. mkdir yo && touch yo/hey.txt)

rm path/to/file.ext ~ sends file to recycle bin

rm -r path/to/folder ~ removes folder

mv path/to/file.ext different/path/file.ext ~ move a file

mv old-filename.ext new-filename.ext ~ can also be used to rename a file

cp path/to/file.ext different/path/file.ext ~ copies a file

&& ~ execute multiple commands at once (ex. mkdir name && touch thatdirname/file.ext)

diskpart ~ access microsofts disk partition utility. type "help" for a list of available commands

                                    Network
                                ~~~~~~~~~~~~~~~

ipconfig ~ show current ip addresses (local in most cases)

ping domain.com ~ pings target domain

tracert domain.com ~ show the network path packets take from you to the target

                                ~~~~~~~~~~~~~~~
                                      GIT
                                ~~~~~~~~~~~~~~~

git init ~ initializes current directory as a local git repository

git remote add origin thetargetGITrepoURL ~ adds the currently selected working directory as a local repo for the target repo on git

git clone url ~ clones git repo into local repo

git status ~ check if there are differences between local files and files on git

git add filename ~ adds file to be commited

git add . ~ adds all modified files to be commited

git reset HEAD "filename" ~ removes the target file from the HEAD and the next commit

git commit -m "comment" ~ commits the current local repo into the head with a comment ( the head is the spot where commited files are stored until you push to the git repo)

git push origin master(or branch name) ~ push changes to git repo

git branch name ~ creates branch with that name on working repo

git branch ~ lists current working branch

git checkout branchname ~ switches working branch to the target branch

git fetch remote ~ puts objects from the remote repo into the FETCH_HEAD

git merge remote ~ merges the FETCH_HEAD with current working local repo

git pull --rebase remote branchname(optional) ~ performs a fetch+merge and integrates current remote branch with local working repo. used just like push. ex. 'git pull --rebase origin master' 'git pull --rebase origin branch1'

git remote -v ~ list remote connections to other repos

git remote rename oldname newname ~ rename current working remote. check remote name with "git remote -v"

git log --oneline origin/master ~ show a log of all the commits for selected branch (ex. git log --oneline master)

                                ~~~~~~~~~~~~~~~
                                     HEROKU
                                ~~~~~~~~~~~~~~~

heroku login ~ allows you to login to your account. windows users may have to do this through the cmd prompt initially (start>run>cmd)

heroku keys:add ~ adds heroku keys to your local PC

heroku git:remote -a project ~ adds git repo as remote. "project" is the subdomain of a heroku url (ex dry-forest-555)

heroku create appname ~ create a heroku app. if no name is specified, one will be generated

git push heroku master ~ pushes the selected master remote git repo onto the heroku app you created 

                                ~~~~~~~~~~~~~~~
                                   Shortcuts
                                ~~~~~~~~~~~~~~~

Ctrl + U: Clears the line from the cursor point back to the beginning
Ctrl + A: Moves the cursor to the beginning of the line
Ctrl + E: Moves the cursor to the end of the line
Ctrl + R: Allows you to search through the previous commands
Ctrl + C: Kill the currently executed command and regain control
